めんたるサロン｜会員サイト構想（要件整理）

目的：会員が自分の事業データを入力 → 蓄積 → グラフ/表で可視化。
管理者（小川さん）は全会員データを一覧把握し、成長状況・課題を見える化する。

1. 主要要件（今回のゴール）

会員側（入力・閲覧）

  • 会員ログイン（会員限定）
  • 自分の事業データを入力できる（フォーム入力）
  • 入力したデータを蓄積し、推移をグラフ/表で見れる

管理者側（全体把握）

  • 全会員のデータを一覧で見れる（表・フィルタ）
  • 会員ごとの履歴ページを見れる
  • 入力状況（入力済/未入力）を把握できる

※将来的に 自動連携（広告/SNS/解析） や AIによる改善提案 を追加して 「オンラインサロン」→「経営インフラ」へ拡張する想定。

2. 会員が入力するデータ例

売上 / 商品

  • 月次売上（総売上）
  • 商品情報（商品名、単価、粗利、成約数 など）

集客

  • のべ集客数
  • 新規集客数

広告

  • 広告費
  • CPA / CPL / ROAS（わかる範囲で）

SNS

  • フォロワー数（増減）
  • 表示回数 / リーチ / 投稿数（任意）

入力形式の優先順位：
①フォーム入力（最優先） → ②CSVアップロード（上級者向け） → ③API連携（拡張フェーズ）

3. 可視化（会員側ダッシュボードのイメージ）

  • 月別推移：売上 / 新規集客 / 広告費 / フォロワー増減
  • 期間切替：今月・先月・過去3ヶ月・過去1年
  • 表：月次の入力履歴（編集・追記）
推移グラフ 月次テーブル 期間フィルタ

4. 管理者画面（小川さん用）

  • 会員一覧（検索・並び替え）
  • 全会員の月次データ一覧（表で一括把握）
  • 入力未実施者の抽出（リマインド対象）
  • ランキング（売上伸び率、広告効率、SNS伸び率など）
  • 異常値の発見（広告費だけ急増、売上急落など）

管理者権限：admin / 会員権限：member のようにロール分離する想定。

5. 実装パターン（選択肢）

パターンA：WordPress（最速）

  • 短期で立ち上げやすい
  • ただし分析・拡張で限界が来やすい

パターンB：Webアプリ（理想）

  • Next.js + DB（例：Supabase）
  • 管理者ダッシュボード/分析の自由度が高い
  • 将来のAI示唆・API連携に強い

MVP（最小構成）で始めるなら：
会員ログイン → 月次入力（売上/新規/広告/SNS） → 会員の推移表示 → 管理者の全員一覧

6. 次に決めると失敗しにくい項目

  • 会員数の想定（初月〜6ヶ月）
  • 入力頻度（週次 or 月次）
  • 決済の位置づけ（外部で課金して紐付ける / サイト内で完結）
